Inside the first layer of a CNN

Build a filter
by hand

A filter (also called a kernel) is just a small grid of numbers, here three by three. The image classifier from the first demo is built almost entirely from one operation repeated: slide that filter across the image and, at each spot, multiply its nine numbers by the pixels underneath and add them up. That's a convolution. Below is the actual code doing it. Change the numbers in the filter (or edit the loop itself), press run, and watch which features light up. A real CNN does exactly this, except it learns the numbers instead of you choosing them.

Choose an image

The filter runs on the brightness of each pixel. Anything with clear edges works well.

The filter, and the code that runs it

Each of the nine weights says how much the matching neighbouring pixel counts; the centre weight is the pixel itself. Load a preset, then edit the numbers and watch the output change.

These presets are classic filters, hand-designed by image scientists decades before deep learning to pick out edges, blur, or sharpen. Several are well known enough to have names (Sobel, Laplacian). A CNN uses the very same 3×3 machinery, but instead of a person choosing the numbers, it learns them from data.

      Go deeper: from one filter to a whole network
      

        
What you are editing is a single convolutional filter (also called a kernel). Sliding it over the image produces one feature map, a new image that is bright wherever the filter's pattern was found. A real convolutional layer holds dozens or hundreds of these filters running in parallel, each producing its own feature map: one for vertical edges, one for a particular texture, one for a blob of a certain colour.

        
Two more ingredients turn this into a network. After each convolution comes a simple non-linearity, usually ReLU (anything negative becomes zero), which keeps only the "this pattern is present" signal. Then pooling shrinks each feature map, so the next layer's filters see a larger area and can combine simple parts into bigger ones: edges into corners, corners into eyes, eyes into faces. ResNet-50, the network in the classifier demo, stacks this pattern 50 layers deep.

        
The crucial difference from what you did here: you chose the nine numbers by hand. A CNN starts with random numbers and learns them, nudging every weight in every filter, layer by layer, from millions of labelled examples until the whole stack lands on the right label. Nobody tells it "detect edges first", it discovers that edge detectors are useful, exactly the kind of edge-detecting filter you built by hand in this demo.
